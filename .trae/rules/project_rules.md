# Open-PU Project Rules

## 开发规则

### 1. 代码修改后的检查流程
- ✅ **每次修改代码后，必须自己检查是否改好**
- ✅ 启动本地服务器测试功能
- ✅ 验证修改是否达到预期效果
- ✅ 确保没有引入新的bug

### 2. Git提交规则
- ✅ **每次修改完代码，必须执行 `git add`, `git commit`, `git push`**
- ✅ 提交信息要清晰描述修改内容
- ✅ 确保所有改动都及时同步到远程仓库

### 3. UI修改验证
- ✅ 检查CSS样式是否正确应用
- ✅ 验证动画效果是否流畅
- ✅ 测试交互功能是否正常
- ✅ 确保响应式设计在不同屏幕尺寸下正常工作

### 4. Bug修复验证
- ✅ 重现bug并确认问题
- ✅ 实施修复方案
- ✅ **验证修复是否有效**
- ✅ 确保没有引入新问题

### 5. 功能测试清单
- [ ] 教程系统是否正常工作
- [ ] 段落库拖拽是否流畅
- [ ] 总谱编辑是否正常
- [ ] 导出功能是否正常
- [ ] 所有按钮是否可点击
- [ ] 模态框是否正常打开/关闭

## 项目上下文

- **项目名称**: Open-PU
- **项目类型**: Web-based music score editor
- **开发者**: 业余贝斯手 + AI工程师
- **技术栈**: HTML, CSS, JavaScript, Tailwind CSS
- **部署**: GitHub Pages

## 设计原则

- 深色主题设计
- 现代化UI交互
- 流畅的动画效果
- 用户友好的体验
